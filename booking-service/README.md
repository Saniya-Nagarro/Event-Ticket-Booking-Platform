# Booking Service

Booking Service for the Event Ticket Booking Platform.

## Features

- Customer can book tickets
- Customer can view own bookings
- Customer can cancel own booking
- Admin/Super Admin can view all bookings
- JWT validation using token generated by User Service
- Calls Event Service to reduce/increase seats
- Publishes Kafka events:
  - `BOOKING_CREATED`
  - `BOOKING_CANCELLED`

## Port

```text
8083
```

## Database

Create database:

```sql
CREATE DATABASE booking_service_db;
```

Default PostgreSQL config:

```yaml
jdbc:postgresql://localhost:5433/booking_service_db
username: postgres
password: root
```

## Kafka

Kafka broker expected at:

```text
localhost:9092
```

Topic used:

```text
booking-events
```

## APIs

### Book Ticket

```http
POST /api/bookings
Authorization: Bearer <CUSTOMER_TOKEN>
Content-Type: application/json
```

```json
{
  "eventId": 1,
  "numberOfTickets": 2
}
```

### My Bookings

```http
GET /api/bookings/my
Authorization: Bearer <CUSTOMER_TOKEN>
```

### Cancel Booking

```http
PUT /api/bookings/{id}/cancel
Authorization: Bearer <CUSTOMER_TOKEN>
```

### Admin View All Bookings

```http
GET /api/bookings/admin/all
Authorization: Bearer <ADMIN_OR_SUPER_ADMIN_TOKEN>
```

## Important

The `jwt.secret` must be same in User Service, Event Service, and Booking Service.

Booking Service forwards the incoming JWT token to Event Service for reduce/increase seat APIs.
